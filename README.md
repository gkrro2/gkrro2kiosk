# 학습개발2팀 키오스크 — 관리 안내

매월 생일자 축하 커피 주문을 모으고 집계하는 팀 내부 웹앱입니다.
이 문서는 **현재 관리자를 위한 평상시 관리 안내**입니다.

---

## 시스템 구조

| 조각 | 역할 | 위치 |
|---|---|---|
| 깃허브 저장소 | 화면 파일(index.html) 보관 + 팀 공개(GitHub Pages) | 관리자 본인의 깃허브 계정 |
| 앱스 스크립트 | 데이터 저장/불러오기 서버 (Code.gs) | 관리자 본인의 구글 계정 |
| 스프레드시트 "학습개발2팀 키오스크 데이터" | 데이터 저장소 (자동 생성) | 관리자 본인의 구글 드라이브 |

계정은 모두 **관리자 본인 것**입니다. 누구에게도 계정을 넘기지 않습니다.

---

## 매달 하는 일

1. 앱의 **카페 관리** 탭(관리자 비밀번호)에서 이번 달 카페 지정. 여러 번 마시면 차수 추가
2. 팀원들에게 "주문하세요" 공지 → 각자 앱에서 이름 선택 후 주문 (연차인 사람은 불참 버튼)
3. **집계** 탭에서 "주문 내용 복사" → 카페에 전달
4. 생일자 축하 카드는 상단 배너에서 팀원들이 자유롭게 작성

집계는 구글 시트의 **대시보드 / 주문내역** 탭에도 자동 정리됩니다.

⚠️ 시트의 `_data` 탭은 앱의 원본 데이터입니다. **절대 직접 수정 금지.**
다른 정리 탭들도 보기 전용입니다(직접 고쳐도 다음 저장 때 덮어써짐). 모든 수정은 앱에서.

---

## 팀원·메뉴 관리 (코드 수정 불필요)

- 팀원 추가/이름·생일 수정/휴직·복직/삭제 → **팀원 관리** 탭
- 카페 추가, 메뉴·가격·옵션 수정 → **카페 관리** 탭 (메뉴 클릭하면 수정)
- 잘못 들어간 주문 삭제 → **집계** 탭에서 이름 옆 × (관리자 모드)

---

## 기능 수정이 필요할 때 — 클로드(Claude) 활용

1. 이 저장소에서 **index.html 다운로드**
2. claude.ai(본인 계정) 새 대화에 업로드 후 요청:
   > "우리 팀 생일 커피 주문 키오스크야 (리액트, 깃허브 페이지 호스팅, 구글 앱스 스크립트 저장). ○○를 수정해줘."
3. 받은 index.html을 이 저장소에 **덮어쓰기 업로드** → 1~2분 뒤 반영 (안 보이면 Ctrl+Shift+R)

새 카페를 통째로 추가할 땐 메뉴판 사진을 함께 올리면 메뉴·가격을 추출해서 넣어줍니다.

## 서버(Code.gs) 수정 시 철칙

script.google.com → 프로젝트 → Code.gs 수정 후, **반드시**:

> 배포 → **배포 관리** → 연필 아이콘 → 버전: **"새 버전"** 선택 → 배포

- ⚠️ "새 버전"을 안 고르면 코드를 아무리 고쳐도 반영되지 않습니다
- ⚠️ "새 배포"를 누르면 URL이 바뀌어 앱이 끊깁니다. 반드시 "배포 관리"에서
- 배포 설정 기준: 유형 **웹 앱** / 실행 **나** / 액세스 **모든 사용자**

## 문제 해결

| 증상 | 해결 |
|---|---|
| 화면이 흰색 | Ctrl+Shift+R 강력 새로고침 → 시크릿 창 → F12 콘솔 오류 확인 |
| "저장 실패" | index.html의 API_URL 뒤에 `?action=load` 붙여 접속해 진단: 로그인 화면 → 액세스를 "모든 사용자"로 / 오류 문구 → 새 버전 재배포 |
| 코드 반영 안 됨 | 화면은 깃허브 재업로드, 서버는 새 버전 재배포 |
| 시트 정리 탭 안 바뀜 | 앱스 스크립트에서 함수 `수동갱신` 실행 |

- API 주소 확인: index.html에서 `API_URL` 검색
- 관리자 비밀번호 확인/변경: index.html에서 `ADMIN_PIN` 검색 → 변경 후 재업로드

## 관리자 교체 시

**계정은 아무것도 넘기지 않습니다.** 후임자에게 파일 3개만 전달하면 됩니다:

> **index.html + Code.gs + 설치-매뉴얼** (모두 이 저장소에 있음)

후임자는 설치-매뉴얼을 따라 **본인의 구글·깃허브 계정으로 새로 설치**합니다 (약 30분).
설치가 끝나면 후임자가 팀에 새 주소를 공지하고, 전임자는 본인의 저장소·앱스 스크립트를 삭제하면 끝입니다.
